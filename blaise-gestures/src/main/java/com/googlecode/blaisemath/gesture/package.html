The Gesture API

The gesture API provides advanced mouse handling support. {@code GestureOrchestrator} maintains a registry of {@code MouseGesture}s, allowing clients to intelligently pass along mouse events to appropriate gestures.

In addition to receiving mouse events, gestures support a lifecycle of activation and use:

  • Gestures are registered with the orchestrator in priority.
  • Gestures determine the conditions under which they may be activated.
  • Gestures are activated when these conditions are met, and they have priority over other gestures.
  • When active, gestures are passed mouse events from the owning component.
  • Active gestures may be canceled, which resets their state and inactivates them.
  • Active gestures may be finalized, which completes the gesture and takes appropriate action.

In Practice

In practice, a {@code LayerUI} is used to setup mouse handling for the component owning an orchestrator, using {@link com.googlecode.blaisemath.gesture.GestureLayerUI}. If there is no active gesture, the set of gestures registered with the orchestrator are given the opportunity to "claim" mouse events and activate themselves (in order of priority). Once a gesture has been activated, it receives all mouse events until it is completed or canceled.
